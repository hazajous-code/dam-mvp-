# 개발사 평가 · 연동 가능 범위 체크리스트 (1페이지)

> 목적: 본 MVP(DAM 고도화 데모)를 실제 시스템으로 구현할 때 **"어디까지 가능한가"**를 개발사와 빠르게 확정하기 위한 점검표.
> 전제: UI·업무 흐름·자동화 개념은 MVP에서 이미 검증됨. **변수는 거의 전부 "원천 시스템 연동"과 "룰 코드화"에 있음.**

데모 라이브: https://hazajous-code.github.io/dam-mvp-/ — 헤더의 **🔌 연동 보기** 토글을 켜면 각 화면의 실제 연동 포인트가 주석으로 표시됩니다.

---

## 0. 첫 미팅 30분 안에 확인할 1가지
**"아래 5개 시스템이 각각 무엇을 API로 노출하는가?"** — 이 답이 곧 가능 범위의 지도입니다.
API가 없거나 닫혀 있으면 그 영역은 "자동"이 아니라 "수기 입력 + 검증"까지만 가능합니다.

---

## 1. API 인벤토리 양식 (개발사/시스템 오너가 채움)

| # | 시스템 | 읽기(Read) | 쓰기(Write) | 인증 방식 | 실시간/배치 | 레이트리밋 | 비고 |
|---|--------|:---:|:---:|---------|---------|---------|------|
| 1 | **PIM2.0** (기준정보) | ☐ | ☐ | | | | 카테고리 LV1~4·모델·SKU·Locale |
| 2 | **AEM Assets Cloud** (DAM) | ☐ | ☐ | | | | 폴더 생성·메타 쓰기·자산 이관 |
| 3 | **CMS/WCM (AEM Pages)** | ☐ | ☐ | | | | Edit/Live URL·컴포넌트·이미지 상태 |
| 4 | **GP1 Admin / 채널** | ☐ | ☐ | | | | LG.com·Criteo·Retailer·Contents Hub 배포 상태 |
| 5 | **SSO / IdP** | ☐ | — | | | | HQ·BU·Agency·외부사 계정/조직 구분 |

> ☐ = 가능 여부 체크. "쓰기 불가"인 시스템은 자동 실행이 막히므로 우선 표시.

---

## 2. 시스템별 핵심 질문

### PIM2.0
- [ ] 카테고리(BU=LV1 ~ LV4)·모델·SKU·Locale 기준정보를 **읽기 API**로 받을 수 있나?
- [ ] 갱신 주기는? (실시간 / 일배치) 변경 감지(webhook) 가능?
- [ ] 메타데이터 매핑의 "정본(Source of Truth)"은 PIM인가?

### AEM Assets Cloud (DAM)
- [ ] 표준 폴더 트리 **생성 API**가 있나? (BU/Category/Model/Channel/Locale)
- [ ] 자산 업로드·메타데이터 쓰기·**이관(Transfer) API**가 있나?
- [ ] Archive Rule(대체/병존/신규)을 시스템이 지원하나, 아니면 우리가 구현?
- [ ] 보존·삭제(이관일+60일 알림) 정책을 스케줄러로 걸 수 있나?

### CMS/WCM (AEM Pages)
- [ ] Edit/Live Page URL·컴포넌트·이미지·CTA 상태를 **조회**할 수 있나?
- [ ] PIM/Admin 값 ↔ AEM 반영값 **대조**가 API로 가능한가?
- [ ] 이미지 깨짐·404·Zoom/360 존재 여부를 자동 점검할 수 있나?

### GP1 Admin / 채널
- [ ] 채널별 배포 상태/이력을 받아올 수 있나?
- [ ] Contents Hub 공유·다운로드 이력 조회 가능?

### SSO / IdP / 권한
- [ ] HQ·BU·Agency·Librarian·CNX QA·외부 개발사를 어떤 ID 체계로 구분?
- [ ] 역할별 권한(생성·승인·이관)을 어디서 관리? (앱 내 / IdP 그룹)

---

## 3. 룰 코드화 가능성 (자동화율을 결정)

- [ ] 데모 룰 카탈로그(폴더경로·메타·파일명·채널·Locale·Archive·WCM QA)에 대응하는 **실제 운영 룰 전체**를 수집할 수 있나?
- [ ] 각 룰의 **Severity(Critical/Warning/Info)** 와 **자동승인/검토/차단** 기준에 합의된 오너가 있나?
- [ ] "사람이 반드시 봐야 하는 예외"의 경계가 명확한가? (모호하면 자동화율 ↓)
- [ ] 룰 변경 권한·승인 절차·감사 이력 요건은?

---

## 4. 거버넌스 / 비기능 요건

- [ ] **보존·삭제 정책**: "이관일 + 60일 삭제 알림"의 정책 오너는? 알림 채널(Email/Slack/Teams)?
- [ ] **이력/감사**(JIRA 대체): 기록 범위·조회·내보내기 요건?
- [ ] **알림**: 어떤 이벤트(승인 요청·반려·이관 완료·삭제 임박)를 누구에게?
- [ ] **규모**: 연 프로젝트 수 / 자산 수 / 동시 사용자 / Locale 수 → 아키텍처·비용
- [ ] **보안**: 외부 개발사·Agency 접근 범위, 자산 다운로드 통제

---

## 5. 개발사에 요청할 산출물

1. **§1 API 인벤토리** 작성본 (엔드포인트·인증·읽기/쓰기·레이트리밋)
2. **PIM2.0 ↔ AEM Assets 메타데이터 매핑표** (필수전송/참고/자동/추천/이관확정 구분)
3. **연동 스파이크(PoC) 1~2주**: 2~3개 시스템을 실제로 읽고/쓰는지 검증
4. **단계별 제안서**:
   - **Phase 1 — 읽기·검증 중심**: PIM 자동입력 + Gatekeeping 자동 검증(읽기) + 대시보드
   - **Phase 2 — 쓰기·이관**: AEM 폴더 생성·메타 쓰기·자산 이관 자동화
   - **Phase 3 — QA·운영**: WCM QA 자동 점검 + 알림 + 감사 이력 + 보존/삭제 스케줄러

## 6. 데모를 미팅에서 쓰는 법
화면을 띄우고 **🔌 연동 보기**를 켠 뒤, 각 화면의 주석을 보며 *"이 동작이 실제로는 어느 API로 대체되는가?"* 를 하나씩 확인하세요. 추상적 논쟁 없이 **연동 가능/불가 → 가능 범위**가 바로 도출됩니다.

| 데모 화면 | 미팅에서 던질 질문 |
|---|---|
| Create Project | PIM2.0 기준정보를 읽어 자동 채울 수 있나? 폴더 생성은 AEM API로? |
| Asset Upload | AEM Assets 업로드 API + 룰 엔진 검증이 가능한가? |
| Gatekeeping QA | 룰을 결정론적으로 코드화 가능한가? 예외 라우팅은? |
| WCM QA | AEM Pages 상태·이미지 헬스체크를 자동화할 수 있나? |
| Transfer & Approval | 이관 API + 채널 배포 + 보존/삭제 스케줄러가 되나? |
| Rule Settings | 룰 스토어·변경 이력·런타임 평가 구조가 되나? |
