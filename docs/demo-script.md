# DAM 고도화 MVP — 데모/발표 스크립트

> 대상: 광고주 · 개발사 · 내부 의사결정자

> 라이브: https://hazajous-code.github.io/dam-mvp-/
> 표기: 〔멘트〕= 말할 내용, **[클릭]** = 화면 조작, 💡 = 강조 포인트

## 0. 오프닝 (30초)

〔멘트〕
"지금 보시는 것은 **LG전자 글로벌 DAM 운영을 고도화하는 방향을 보여주는 데모**입니다.
현재 제품 이미지·영상 같은 '자산'을 만들어 등록하는 과정은 PIM·DAM·CMS·WCM·수기 QA가 따로 놀아서, 폴더 만들고 메타 입력하고 검수하는 일을 사람이 반복합니다.
이 데모는 그 과정을 **한 화면에 모으고, 단순 확인은 자동화하고, 사람은 예외만 판단하게** 만들면 어떤 모습인지 보여줍니다.
실제 시스템 연동 없이 샘플 데이터로 동작하지만, **업무 흐름과 자동화 구조**는 그대로 담았습니다."

💡 핵심 메시지 3개를 먼저 심기: ① 한 곳에 모음 ② 정상은 자동·예외만 사람 ③ 계정별 자동입력·추천

〔멘트〕
"먼저 전체 그림입니다. 가로는 **5단계** — ①기준정보·생성 → ②제작·업로드 → ③자동 검수 → ④품질·이관 → ⑤운영·분석.
세로는 **역할 레인** — HQ PMO, Agency(제작사), System(자동화), CNX QA, Librarian(사서).

보라색 점선 카드는 **사람이 아니라 시스템이 자동으로 처리**하는 단계예요.
아래 단계마다 프로젝트 '상태값'이 매핑돼 있고, 하단 각주는 **실제로 연결될 시스템**(PIM2.0·AEM Assets·AEM Pages·GP1·보존정책)입니다."

💡 "오늘 데모는 이 5단계를 왼쪽에서 오른쪽으로 따라가며 보여드리겠습니다."
💡 카드를 클릭하면 해당 실제 화면으로 이동한다는 점을 시연(예: '프로젝트 생성' 클릭).

## 4. 시나리오 1 — 프로젝트 생성 (HQ PMO) (3분)

**[클릭]** 역할 = **HQ PMO** 확인 → 좌측 **Create Project**

〔멘트〕
"새 자산 프로젝트를 만듭니다. 여기가 이 시스템의 가장 똑똑한 부분입니다."

### 4-1. 계정별 자동 입력
〔멘트〕
"상단을 보세요. 지금 로그인한 **김민준(HQ PMO)** 계정의 평소 성향을 알고 있어서, 본부(MS)·채널(LG.com)·언어(en-US)·선호 템플릿(Product PDP)을 **미리 채워** 놨습니다."

**[클릭]** 상단 '계정 기준(데모)'를 **윤채원(Agency)** 으로 변경

〔멘트〕
"계정을 바꾸면 기본값이 바로 달라집니다 — 본부 HS, 언어 de-DE, 선호 템플릿이 Gallery로요.
즉 **누가 만드냐에 따라 자동으로 맞춰 채워주는** 구조입니다." (다시 김민준으로 돌려놓기)

### 4-2. PIM 카테고리 (BU=LV1 ~ LV4)
**[클릭]** Step1에서 Project Name 입력, BU(=LV1) 확인, **LV2 → LV3 → LV4** 순서로 선택 (예: Home Entertainment → TV → OLED evo G5), Model 입력

〔멘트〕
"카테고리는 PIM 기준으로 **BU가 1단계(LV1)**, 그 아래 LV2·LV3·LV4로 좁혀집니다. 실제로는 이 목록을 PIM2.0에서 받아옵니다."

### 4-3. 템플릿 추천
**[클릭]** Step3(템플릿 선택)까지 '다음'

〔멘트〕
"상단에 **AI 추천**이 뜹니다. 채널·카테고리·계정 선호를 종합해 **신뢰도(%)와 근거**까지 제시하고, 추천 템플릿에는 '추천' 배지가 붙습니다. '추천 적용' 한 번이면 됩니다."

### 4-4. 메타데이터 매핑 + 목표 이관일
**[클릭]** Step4 → '추천값 적용', Step5(생성)로 이동

〔멘트〕
"4단계는 메타데이터를 **PIM에서 온 값 / 사용자 입력 / DAM 필수 전송 값 / 계정 기반 추천 값**으로 구분해 보여줍니다.
마지막 단계에 **Transfer Target은 '목표 이관일' 날짜**로 정하는데, 여기 주의문구가 있습니다 — 💡**'설정한 날짜로부터 60일 이후 프로젝트 삭제 알림이 발송될 수 있음'**. 방치 자산을 막는 보존 정책입니다."

**[클릭]** '프로젝트 생성' → 생성된 프로젝트 상세로 이동

〔멘트〕
"생성하면 예상 폴더 경로와 함께 프로젝트가 만들어지고, 이력에 '계정 기본값 + 추천 템플릿 적용'이 기록됩니다."

**[클릭]** 좌측 **Gatekeeping QA** → 프로젝트 선택기에서 정상 건(예: PRJ-2603) 선택

〔멘트〕

이 프로젝트는 룰을 다 통과해서 **Auto Approved** — 담당자 확인 없이 자동 승인됩니다. 룰별 Pass/Warning 결과가 표로 보입니다."

**[클릭]** 프로젝트 선택기에서 **PRJ-2605 (ES Air Purifier B2B)** 로 변경

〔멘트〕
"이번엔 예외 건입니다. 자동 판정이 **Exception Required**로 떨어졌죠.
이유는 아래 '예외 사유 상세'에 — 목표 이관일 누락 같은 **Critical 위반**이 있어서입니다.
💡 정상 건은 그냥 통과시키고, **이렇게 진짜 문제가 있는 것만 사람에게** 올라옵니다. 담당자는 코멘트를 달고 승인 또는 반려합니다."

**[클릭]** (HQ PMO/Librarian 권한으로) 코멘트 입력 후 '예외 승인' 시연

〔멘트〕

## 13. 예상 Q&A

- **Q. 실제로 동작하나요?** A. 지금은 mock·localStorage 데모입니다. 흐름·UI·자동화 로직은 실제와 동일하게 구현했고, 연동만 붙이면 됩니다.
- **Q. 자동 승인이 위험하지 않나요?** A. Critical 룰을 통과한 정상 건만 자동 승인하고, 하나라도 위반하면 사람에게 올립니다. '자동 승인 허용 범위'는 거버넌스로 합의합니다.
- **Q. 추천은 AI인가요?** A. 현재는 규칙+계정 선호 기반의 설명 가능한 추천입니다. 필요하면 사용 이력 학습으로 고도화할 수 있습니다.
- **Q. 기존 시스템을 다 바꿔야 하나요?** A. 아니요. 이 시스템은 그 위에 얹는 '오케스트레이션' 층입니다. 각 시스템의 API만 연동합니다.
